## 개요

프롬프트 엔지니어링은 AI 모델로부터 원하는 결과를 얻기 위해 입력 텍스트(프롬프트)를 체계적으로 설계하는 기법입니다. 이 포스트에서는 4가지 핵심 기법을 실습합니다.

- **Role Playing**: 페르소나 설정
- **Few-Shot Learning**: 예시 기반 학습
- **Chain of Thought**: 단계별 사고
- **MultiModal**: 이미지 + 텍스트 분석

---

## 1. Role Playing (페르소나 설정)

AI에게 특정한 역할이나 성격을 부여하여, 그 역할에 맞는 방식으로 응답하도록 하는 기법입니다.

### 핵심 요소

- **역할 정의**: "너는 ~이야" 형태로 명확한 역할 부여
- **성격/톤**: 친근함, 전문성, 유머 등 원하는 톤 설정
- **행동 지침**: 어떻게 응답해야 하는지 구체적 가이드라인 제시

### 실습 코드

```python
system_prompt = '''
너는 이제 친근한 과학자 선생님이야.
아이들이 과학에 흥미를 가질 수 있도록 재미있는 실험이나 예시를 들어서 설명해줘.
어려운 용어는 쉬운 말로 바꿔서 설명하고, 호기심을 자극하는 질문도 함께 던져줘.
'''

user_prompt = '왜 하늘은 파란색일까요?'

messages = [
    ("system", system_prompt),
    ("user", user_prompt)
]

result = llm.invoke(messages)
```

### 실행 결과

AI가 "친근한 과학자 선생님" 역할로 레이리 산란 현상을 아이들 눈높이에 맞춰 설명합니다:

> 하늘이 파란 이유는 **햇빛이 공기 속에서 특별한 방식으로 흩어지기 때문**이에요.
> 햇빛은 사실 빨·주·노·초·파·남·보 여러 색이 섞인 빛이에요.
> 파란색 빛은 파장이 짧아서 공기 분자에 잘 부딪히고, 여기저기 **많이 흩어져요.**
> 이 현상을 **'레이리 산란'**이라고 부릅니다.

## 2. Few-Shot Learning (예시 기반 학습)

AI에게 몇 개의 구체적인 예시를 제공하여, 그 패턴을 학습하고 비슷한 형태로 응답하도록 하는 기법입니다.

### 핵심 요소

- **예시 제공**: 원하는 답변 형태의 구체적인 예시 2-3개 제시
- **패턴 학습**: AI가 예시의 구조와 스타일을 파악하도록 유도
- **일관성**: 모든 예시가 동일한 형식과 품질을 유지

### 활용 시나리오

- 특정 형식의 답변이 필요할 때
- 일관된 스타일의 콘텐츠를 생성할 때
- 복잡한 구조의 정보를 정리할 때

### 실습 코드

```python
system_prompt = '''
다음 예시를 참고해서 동물에 대해 설명해줘:

예시 1:
동물: 고양이
특징: 털이 부드럽고 야옹야옹 울어요
서식지: 집에서 사람과 함께 살아요
재미있는 점: 높은 곳을 좋아하고 쥐를 잡아요

예시 2:
동물: 강아지
특징: 꼬리를 흔들고 멍멍 짖어요
서식지: 집에서 사람과 함께 살아요
재미있는 점: 산책을 좋아하고 주인에게 충성해요
'''

user_prompt = '코끼리에 대해 설명해줘.'
messages = [("system", system_prompt), ("user", user_prompt)]
result = llm.invoke(messages)
```

### 실행 결과

AI가 예시와 동일한 형식으로 응답합니다:

> 동물: 코끼리
> 특징: 몸집이 크고 기다란 코와 큰 귀가 있어요. 두꺼운 회색 피부를 가지고 있어요.
> 서식지: 아프리카 사바나와 숲, 아시아의 정글과 초원에서 무리를 지어 살아요.
> 재미있는 점: 코로 물을 뿜어 샤워를 하고, 코를 손처럼 써서 물건을 집어요.

## 3. Chain of Thought (단계별 사고)

AI가 복잡한 문제를 단계별로 차근차근 해결하도록 유도하는 기법입니다.

### 핵심 요소

- **단계별 분해**: 복잡한 문제를 작은 단계로 나누기
- **사고 과정 표시**: 각 단계의 추론 과정을 명시적으로 보여주기
- **논리적 연결**: 각 단계가 논리적으로 연결되도록 구성

### 활용 시나리오

- 수학 문제나 논리 문제를 풀 때
- 복잡한 분석이나 추론이 필요할 때
- 결론에 이르는 과정을 투명하게 보여주고 싶을 때

### 실습 코드

```python
system_prompt = '''
당신은 단계별 사고를 통해 문제를 해결하는 AI 어시스턴트입니다.
문제를 받으면 다음과 같이 응답하세요:

1. 문제 파악: 무엇을 묻고 있는지 명확히 하기
2. 단계별 분석: 해결 과정을 차근차근 설명
3. 최종 답안: 결론 제시

각 단계를 명확히 구분하여 설명해주세요.
'''

user_prompt = '''
사라는 사과 15개를 가지고 있었습니다.
친구들에게 3개씩 4명에게 나누어 주었습니다.
그 후 엄마가 사과 7개를 더 주셨습니다.
사라가 최종적으로 가진 사과는 몇 개인가요?
'''

messages = [("system", system_prompt), ("user", user_prompt)]
result = llm.invoke(messages)
```

### 실행 결과

AI가 단계별로 풀이 과정을 보여줍니다:

> **1. 문제 파악**: 처음 15개 → 친구 4명에게 3개씩 나눠줌 → 엄마가 7개 추가
>
> **2. 단계별 분석**:
> - 처음 사과: 15개
> - 나누어 준 사과: 3 × 4 = 12개
> - 남은 사과: 15 - 12 = 3개
> - 엄마가 준 사과 추가: 3 + 7 = 10개
>
> **3. 최종 답안**: 사라가 최종적으로 가진 사과는 **10개**입니다.

## 4. MultiModal (멀티모달 - 이미지 + 텍스트)

텍스트뿐만 아니라 이미지, 오디오 등 다양한 형태의 데이터를 함께 처리할 수 있는 AI 기능입니다.

### 핵심 요소

- **이미지 인코딩**: 이미지를 Base64 형태로 변환
- **멀티모달 프롬프트**: 이미지와 텍스트를 함께 포함하는 메시지 구성
- **비전 모델**: 이미지를 이해할 수 있는 GPT-4V 같은 모델 사용

### 활용 분야

- 이미지 분석 및 설명
- 문서나 차트 해석
- 의료 영상 분석
- 예술 작품 감상 및 분석

### 실습 코드

```python
import base64

def encode_image(image_path):
    with open(image_path, "rb") as image_file:
        return base64.b64encode(image_file.read()).decode('utf-8')

base64_image = encode_image("image/sample_image.png")

system_prompt = '''
이 그림을 분석해주세요. 다음 사항들을 포함해서 설명해주세요:
1. 작가 정보 (화가명, 국적, 활동시기)
2. 작품 정보 (작품명, 제작연도, 크기 및 재료/기법)
3. 미술사적 배경
4. 작품 특징 (화풍, 색채, 구도, 상징적 의미)
5. 의의와 영향
'''

user_prompt = '해당 그림 한국어와 영어로 각각 설명해줘.'

messages = [
    ("system", system_prompt),
    ("user", [
        {"type": "text", "text": user_prompt},
        {
            "type": "image",
            "base64": base64_image,
            "mime_type": "image/png"
        }
    ])
]

result = llm.invoke(messages)
```

### 실행 결과 (반 고흐 - 별이 빛나는 밤)

AI가 이미지를 분석하여 작품 정보를 상세히 제공합니다:

> **작가**: 빈센트 반 고흐 (Vincent van Gogh), 네덜란드, 19세기 후반
> **작품명**: 《별이 빛나는 밤》 (The Starry Night), 1889년
> **기법**: 캔버스에 유채, 약 73.7 × 92.1 cm
>
> 후기 인상주의에 속하며, 임파스토 기법으로 두껍게 물감을 올려 바르고
> 나선형의 붓질이 하늘 전체를 소용돌이치는 에너지로 가득 채웁니다.
> 강렬한 푸른색과 노란색의 대비가 밤하늘의 신비로운 분위기를 만듭니다.

## 프롬프트 엔지니어링 기법 비교

| 기법 | 핵심 원리 | 적합한 상황 | 조합 가능성 |
|------|-----------|-------------|-------------|
| **Role Playing** | AI에게 역할 부여 | 특정 톤/스타일의 응답 필요 시 | Few-Shot + Role Playing |
| **Few-Shot** | 예시로 패턴 학습 | 일관된 형식의 출력 필요 시 | CoT + Few-Shot |
| **Chain of Thought** | 단계별 추론 유도 | 복잡한 문제 해결, 논리적 분석 | Role Playing + CoT |
| **MultiModal** | 이미지+텍스트 결합 | 시각 자료 분석, 문서 해석 | 모든 기법과 조합 가능 |

### 기법 조합 예시

```python
# Role Playing + Few-Shot + CoT 조합
system_prompt = '''
너는 수학 교사야. (Role Playing)

예시: (Few-Shot)
문제: 사과 5개에서 2개를 먹으면?
풀이: 5 - 2 = 3개

모든 문제를 단계별로 풀어줘. (Chain of Thought)
'''
```

이 기법들을 조합하면 더욱 정교한 AI 활용이 가능합니다. 예를 들어, Role Playing으로 전문가 페르소나를 설정하고, Few-Shot으로 출력 형식을 지정하며, CoT로 추론 과정을 투명하게 만들 수 있습니다.

## 정리

- **Role Playing**은 시스템 프롬프트에서 역할을 명확히 정의하여 AI의 응답 스타일을 제어
- **Few-Shot Learning**은 2-3개의 예시를 제공하여 원하는 출력 형식을 학습시킴
- **Chain of Thought**는 "단계별로 설명해줘"라는 지시로 AI의 추론 과정을 투명하게 만듦
- **MultiModal**은 Base64 인코딩된 이미지를 메시지에 포함하여 시각 자료 분석 가능
- 이 기법들을 조합하면 더욱 정교한 AI 활용이 가능
